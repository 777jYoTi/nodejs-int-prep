Q1. What is node.js ?

Ans.

        * Node.js is an open-source JavaScript runtime built on Chrome's V8 engine. 

        * It allows running JavaScript code outside the browser, making it great 
          for server-side applications.

        *  Key features: asynchronous, non-blocking, 
                         NPM for package management, 
                         cross-platform, 
                         ideal for real-time apps, 
                         and command-line tools.

CROSS QUESTIONS ;-

No-1 :- Open-Source Cross-Platform: 

        * Open-source refers to software that comes with its source code freely available to the public. 
          This means that anyone can view, modify, and distribute the code.

        * Cross-platform means that the software can run on multiple operating systems, like
          Windows, macOS, or Linux, without needing major changes.

        * So, when we say "open-source cross-platform," it means the software is freely available,
          and it can work on different types of computers.

No-2 :- Blocking vs. Non-Blocking: 

        *Blocking: When something is blocking, it means that a task or operation makes the program 
                   wait until it's finished before moving on to the next task.
        
        *Non-Blocking: On the other hand, non-blocking means that a task or operation allows 
                       the program to continue with other things while waiting for a certain task to complete.

Q3. What are the modules in Node.js?

Ans.

         * Modules are reusable blocks of code and functionality that use in our project .

         * It can be easily imported and used in other parts of a Node.js application.

         * Three types Modules :- Core Modules: These are built-in modules that come bundled with Node.js.

                                  Local Modules: These are custom modules created by developers.

                                  Third-party Modules: These are external modules developed by the Node.

CROSS QUESTIONS ;-

No-1 :- What is EXPORT, IMPORT, REQUIRE Keyword? 

         * EXPORT is used to make things available for other modules to use.

             EX:
             const myFunction = () => {};
             const myVariable = 42;
            
             module.exports = {
              myFunction,
              myVariable,
            };

         * IMPORT is not a direct keyword but is  using the 
           REQUIRE function to import functionality from other modules.

           EX:
           const myModule = require('./myModule');

           myModule.myFunction();
           console.log(myModule.myVariable);

         * REQUIRE is used to include external modules or libraries into your Node.js code, 
           allowing you to access their functionality.

           EX:
           const fs = require('fs'); Node.js module 'fs'
           const express = require('express'); 'express'

Q5. Difference between default export and named export

Ans.

      * Default Export:
                       A module can have only one default export.
					   
                       It is used when a module wants to export a 
					             single value or function as its default export.
					   
                       When another module imports the default export, 
			                 it can choose to give it any name during the import.
      *  Named Export: 
                      A module can have multiple named exports.
                      
                      It is used when a module wants to export multiple 
					           functions, objects, or variables from the same file, 
					           and each of them can be accessed individually in the 
					           importing module.

Q9. What are the different types of HTTP requests?

Ans. Basicaly we use this requests

        1. GET: Used to retrieve data from a server; a read-only request.

        2. POST: Used to send data to a server to create or update resources.

        3. PUT: Used to update existing resources on the server.

        4. DELETE: Used to remove a resource from the server.
        
        5. PATCH: Used to apply partial modifications to a resource.
